🔀 Proceso de Merge al final de cada etapa
# 1. Cada uno abre un Pull Request en GitHub
# developer-javier/ → main (Jose revisa)
# developer-jose/ → main (Javier revisa)

# 2. Después del merge, ambos actualizan su rama
git checkout developer-javier/
git pull origin main

# 3. Si hay conflictos, resolverlos y commitear
git add .
git commit -m "fix: resolver conflicto en application.properties"
✍️ Convención de commits recomendada
feat(ms-auth): agregar encriptación BCrypt
feat(ms-personajes): WebClient para consultar razas
fix(ms-inventario): corregir validación de peso máximo
docs(ms-combate): agregar Swagger a endpoints
test(ms-misiones): tests unitarios para completar misión
chore: actualizar application.properties con nuevo puerto
⚠️ Reglas para evitar conflictos
• Cada uno trabaja solo en sus microservicios asignados
• Nunca editar archivos del otro sin coordinarse antes
• Si necesitan compartir una clase utilitaria, hablar primero
• Hacer git pull origin main al inicio de cada día
• Commitear seguido — no acumular 3 días de cambios en 1 commit
• El application.properties es territorio de cada uno — no sobreescribir
