📖 Introducción

Esta guía contiene todos los endpoints de los 11 microservicios del proyecto Realm of Valyron, con métodos HTTP, URLs, y ejemplos de JSON para pruebas.

⚡ Inicio Rápido

1. Importar en Postman: Importa el archivo Postman_Collection.json

2. Obtener JWT: Ejecuta POST /api/v1/auth/login para obtener el token

3. Usar el token: Copia el token en la variable {{jwtToken}} de la colección

4. Pruebas en vivo: Haz clic en "Send" en cada request

5. Probar en Swagger: Abre los URLs del navegador para pruebas interactivas

Autenticación

Servicios que requieren JWT:

  • ms-inventario (POST /api/v1/inventarios)
  • ms-combate (POST /api/v1/combates)

Servicios abiertos: Todos los demás no requieren autenticación.

🔐 ms-auth Puerto 8080

Autenticación y generación de tokens JWT

Método Endpoint Descripción
POST /api/v1/auth/register Registrar nuevo usuario
POST /api/v1/auth/login Obtener JWT token

Request: POST /api/v1/auth/register

{ "username": "admin_valyron", "email": "admin@valyron.com", "password": "valyron123", "rol": "ADMIN" }

Request: POST /api/v1/auth/login

{ "username": "admin_valyron", "password": "valyron123" }

✅ Resultado esperado

Login devuelve { "token": "eyJhbGciOi..." }

Copiar el valor del token y pegarlo en variable {{jwtToken}}

🎯 Orden Recomendado de Pruebas

Sigue este orden para evitar errores de dependencias:

  1. ms-auth (8080) → POST /login para obtener JWT token
  2. ms-razas (8081) → Crear y listar razas (base para personajes)
  3. ms-personajes (8082) → Crear personajes con razas existentes
  4. ms-objetos (8083) → Crear objetos (base para inventario)
  5. ms-regiones (8086) → Crear regiones (base para combate)
  6. ms-poderes (8085) → Crear poderes (independiente)
  7. ms-inventario (8084) → Agregar objetos a personajes 🔒
  8. ms-combate (8087) → Iniciar batallas 🔒
  9. ms-misiones (8088) → Crear y completar misiones
  10. ms-gremios (8089) → Crear gremios con guardianes
  11. ms-historial (8090) → Registrar eventos

💡 Tips importantes

✅ Al completar login, copiar token y pegarlo en {{jwtToken}}

✅ Los IDs semilla (1-6) ya existen, puedes pisarlos con PUT

✅ Los nuevos registros reciben ID 7, 8, 9, etc.

✅ Las referencias entre servicios usan IDs (razaId, objetoId, personajeId, etc.)

✅ Para ms-inventario y ms-combate, incluir header de autenticación

📱 Cómo usar Swagger UI

1. Abre cualquier URL de Swagger en el navegador

2. Haz clic en un endpoint para expandirlo

3. Haz clic en "Try it out"

4. Completa el formulario con los valores

5. Haz clic en "Execute" para enviar la solicitud

6. Verás la respuesta en formato JSON

Para endpoints con JWT: Haz clic en el botón 🔒 "Authorize" arriba → ingresa Bearer {token}

📋 Notas Finales

  • Todos los microservicios están en localhost con puertos 8080-8090
  • JSON debe incluir encabezado Content-Type: application/json
  • Los servicios ms-inventario y ms-combate requieren token JWT en el header
  • Base de datos: MySQL con datos semilla pre-cargados (IDs 1-6)
  • Documentación: Cada servicio tiene Swagger UI en /swagger-ui.html
  • Para imprimir como PDF: Ctrl+P → Guardar como PDF
